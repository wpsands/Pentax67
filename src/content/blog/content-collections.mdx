# {frontmatter.title}

Astro Content Collections provide a structured, type-safe way to manage your site's content.

## Defining a Collection

In `src/content.config.ts`, you define collections with a schema:

```ts
import { defineCollection, z } from 'astro:content';
import { glob } from 'astro/loaders';

const blog = defineCollection({
  loader: glob({ pattern: '**/*.mdx', base: './src/content/blog' }),
  schema: z.object({
    title: z.string(),
    description: z.string(),
    pubDate: z.coerce.date(),
    tags: z.array(z.string()).default([]),
  }),
});

export const collections = { blog };
```

## Querying Content

You can query your collections using `getCollection()`:

```astro
---
import { getCollection } from 'astro:content';

const posts = await getCollection('blog');
const sorted = posts.sort((a, b) =>
  b.data.pubDate.valueOf() - a.data.pubDate.valueOf()
);
---
```

<Callout type="info">
  The `getCollection` function returns fully typed data based on your Zod schema.
  You get autocompletion and type checking for all frontmatter fields!
</Callout>

## Benefits

- **Type safety** — Zod schemas validate frontmatter at build time
- **Glob loader** — Flexibly load content from any directory
- **Querying** — Filter, sort, and transform collections easily
- **MDX support** — Render components inside your content

<Callout type="tip">
  Use `z.coerce.date()` for date fields so both string dates and Date objects work seamlessly.
</Callout>

Content Collections are the recommended way to manage structured content in Astro.
